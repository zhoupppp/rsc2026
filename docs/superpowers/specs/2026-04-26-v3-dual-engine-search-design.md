# RSC 金融专家智库 V3 需求规划：双引擎发现模式

## 1. 业务背景与用户痛点
当前 V2 版本的核心痛点是 **数据检索逻辑混乱，未能发挥“RSC 认证专家智库”的高级数据价值**。
目前系统强制将 RSC 高级数据（管理规模、行业、价值标签）作为普通 SAC/AMAC 金融人才数据库（抓取库）的一个附属过滤条件。这导致了：
- **查全率不足**：普通库缺乏高级标签，一旦用户使用“行业”或“管理规模”筛选，大量潜在匹配的非 RSC 人员被错误过滤。
- **特定场景缺失**：
  1. 无法精准定位特定行业/地区的机构、投资者、分析师。
  2. 无法有效监控特定行业/地区投资者/分析师的工作变动与跳槽。
  3. 无法针对线下活动（如路演）邀请特定画像（行业、市值、关注风格）的垂直人群。
  4. 沟通前缺乏足够的高亮特色信息（特色标签、价值标签）。

## 2. 核心改动：双引擎隔离搜索 (Dual-Engine Discovery)
为满足“默认查询为并集，但提供单独查询入口”的需求，我们将搜索系统拆分为 **两个独立的 Tab / 引擎**，各自具备独立的交叉筛选逻辑。

### 2.1 引擎 A：全市场金融人才库 (The Broad Market Engine)
**定位**：查全、查背景、查执业轨迹（跳槽异动监控）。
**数据范围**：SAC + AMAC 全量抓取数据（近 40 万条）。
**筛选条件**：
- 姓名 / 机构名称（基础搜索）
- 执业类型（基金从业、证券分析师等）
- **交叉筛选**：是否为 RSC 已认证用户 (True/False)
**列表展示**：
- 重点展示：执业资格、过往执业轨迹、跳槽异动标识。
- 若为 RSC 用户，打上专属 Badge。

### 2.2 引擎 B：RSC 认证专家智库 (The Precision Expert Engine)
**定位**：查准、查画像、定向邀约（路演、线下活动、深度背调）。
**数据范围**：仅限 `rsc_users` 与 `rsc_orgs` 中已认证的用户数据。
**筛选条件**：
- 姓名 / 机构名称（基础搜索）
- 关注行业 / 赛道
- 机构管理规模 (AUM)
- 关注风格 / 价值标签
- 所在地区
- **交叉筛选**：档案匹配状态（已匹配 SAC/AMAC 档案、待更新）
**列表展示**：
- 重点展示：雷达图能力模型、特色标签（行业、偏好）、机构管理规模、价值分。

## 3. 场景覆盖映射
| 业务场景 | 对应引擎 | 解决方案路径 |
| --- | --- | --- |
| **了解具体行业/地区的机构与专家** | 引擎 B (RSC) | 通过行业、地区、规模筛选器，快速锁定 RSC 认证专家列表。 |
| **监控行业/地区专家的跳槽异动** | 引擎 A (全市场) | 在全市场库中锁定目标机构/人员，通过抓取的履历轨迹对比，标记 `is_outdated`（待更新/已跳槽）。 |
| **线下活动定向邀约特定人群** | 引擎 B (RSC) | 组合筛选“特定行业 + 市值规模 + 关注风格”，导出邀约名单。 |
| **沟通前深度背调与高亮关注信息** | 引擎 B (RSC)详情 | 详情页顶部提取“特色标签”、“价值标签”，以雷达图和高亮卡片呈现，一目了然。 |

## 4. 技术实现思路 (Draft)
1. **前端改造**：
   - 首页分为两个大 Tab：`[全市场金融人才]` 和 `[RSC 认证专家]`。
   - 不同的 Tab 渲染不同的 `FilterBar` 和列表项组件。
2. **后端改造**：
   - `/api/talents/search` (引擎 A)：查询 SAC/AMAC 联合视图，保留 `is_rsc` 筛选参数。
   - 新增 `/api/rsc/experts` (引擎 B)：直接查询 `rsc_users` 与 `rsc_orgs`，支持基于 JSON 的高级标签、AUM、地区、档案状态 (`is_matched`, `is_outdated`) 等多维度复杂查询。
